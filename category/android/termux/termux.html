软件管理

搜索包
pkg search <query>

安装包
pkg install <package>

卸载包
pkg uninstall <package>

重新安装包
pkg reinstall <package>

更新源
pkg update

升级软件包
pkg upgrade

列出可供安装的所有包
pkg list-all

列出已经安装的包
pkg list-installed

显示某个包的详细信息
pkg show <package>

显示某个包的相关文件夹路径
pkg files <package>

若找不到命令可以更新一下试试
apt upgrade

执行完之后再运行
rustc --version
应该可以正常显示版本号了
此时已可以使用rust了如用 cargo new project_name 命令来创建项目, 但在此编译项目和执行项目有权限的问题
假设当前目录不是在~/路径下即home路径下
若是用 rustc source_file_name 来编译单个无引用库的文件则可以编译成功但编译后的文件无法执行, 会遇到 Permission denied , 即无权限
根据查到的资料说是安桌系统禁止运行shared目录下的文件
需要将执行文件放到~/目录下然后运行 chmod +x 执行文件名
若是用 cargo build 等命令来编译则需将整个项目文件夹放到 ~/ 目录下, 然后将整个项目文件夹增加执行权限, chomd +x -R 项目文件夹名称, 才能编译成功因为在编译最终的文件前要先编译库文件, 而库文件若无权限则编译库文件时就失败了进而无法继续编译

另外在项目文件夹下执行 cargo doc --open 是很慢的每一个关联的库都会生成文档并且是递归性的, 会生成大量的文档, 例如代码中仅包含了一个标准输入输出库和随机数库就生成了48MB大小的文档

生成密钥对

install openssh package
pkg install openssh
安装后会自动生成各种类型的蜜钥, 蜜钥文件的路径在
/data/data/com.termux/fils/usr/etc/ssh/
下, 将rsa类型的密钥文件复制到~/.ssh/路径下
cp ~/../usr/etc/ssh/ssh_host_rsa_key.pub ~/.ssh/id_rsa.pub
cp ~/../usr/etc/ssh/ssh_host_rsa_key ~/.ssh/id_rsa
