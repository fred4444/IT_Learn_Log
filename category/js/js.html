DOM中attribute与property的区别

DOM中的property就是js中对象的属性
DOM中的attribute是写在html标签中的属性, 比如 <div style="color : red;" id="id_123" my_custom_attribute_1="attribute1" data-my_custom_attribute_2="attribute2"> 中的style, id, my_custom_attribute_1, data-my_custom_attribute_2都属于attribute并且这4个attribute方在DOM对象的attributes子属性对象下可以按attribute名称获取也可以按数组下标获取, 其顺序同写在DOM标签中的顺序, 每个元素类型为AttrPrototype, DOM对象的attributes子属性的值如下:
attributes: NamedNodeMap(4)
0: style="color : red;"
1: id="id_123"
2: my_custom_attribute_1="attribute1"
3: data-my_custom_attribute_2="attribute2"
"data-my_custom_attribute_2": data-my_custom_attribute_2="attribute2"
id: id="id_123"
my_custom_attribute_1: my_custom_attribute_1="attribute1"
style: style="color : red;"
其中的data-my_custom_attribute_2是html5中指定的以data-开头的custom data attributes还在DOM对象的dataset子属性下, 其值如下:
dataset: DOMStringMap(1)
my_custom_attribute_2: "attribute2"
style和id两个attribute是默认的attribute和property共有的所以在DOM对象的子属性中也存在, 而my_custom_attribute_1和data-my_custom_attribute_2则没有, 对于默认的attribute与property共有的attribute，更改 attribute 的值会同步更改DOM对象同名子属性的值, 但更改property的值不会同步到同名attribute中
attribute作为标签中的属性, 其键值均为字符串, 且大小写不敏感, 在用getAttribute获取不存在的attribute的值时返回null
还有一个特殊的attribute与property共有属性href, 用getAttribute('href')则返回标签中写的原值, 但用DOM对象则返回完整的url路径, 例如当前页面路径为"https://example/js.html, 标签为"<a href="./nodejs.html">, 若DOM对象用dom_a表示则, dom_a.getAttribute('href')返回'./nodejs.html', dom_a.href返回"https://example/nodejs.html"

;(function() {

var toString = Object.prototype.toString;

// Used to resolve the decompiled source of functions
var fnToString = Function.prototype.toString;

// Used to detect host constructors (Safari > 4; really typed array specific)
var reHostCtor = /^\[object .+?Constructor\]$/;

// Compile a regexp using a common native method as a template.
// We chose `Object#toString` because there's a good chance it is not being mucked with.
var reNative = RegExp('^' +
// Coerce `Object#toString` to a string
String(toString)
// Escape any special regexp characters
.replace(/[.*+?^${}()|[\]\/\\]/g, '\\$&')

// Replace thing like `for ...` to support environments like Rhino which add extra info
// such as method arity.
.replace(/toString|(function).*?(?=\\\()| for .+?(?=\\\])/g, '$1.*?') + '$'
);

return type == 'function'
// Use `Function#toString` to bypass the value's own `toString` method
// and avoid being faked out.

// Fallback to a host object check because some environments will represent
// things like typed arrays as DOM methods which may not conform to the
// normal native pattern.
: (value && type == 'object' && reHostCtor.test(toString.call(value))) || false;
}

// export however you want
module.exports = isNative;
}());

RegExp instance lastIndex

可参考： RegExpInstance.lastIndex - JavaScript | MDN
The lastIndex is a read/write integer property of regular expression instances that specifies the index at which to start the next match.
Description
This property is set only if the regular expression instance used the g flag to indicate a global search, or the y flag to indicate a sticky search. The following rules apply:
If lastIndex is greater than the length of the string, test() and exec() fail, then lastIndex is set to 0.
If lastIndex is equal to or less than the length of the string and if the regular expression matches the empty string, then the regular expression matches input starting from lastIndex.
If lastIndex is equal to the length of the string and if the regular expression does not match the empty string, then the regular expression mismatches input, and lastIndex is reset to 0.
Otherwise, lastIndex is set to the next position following the most recent match.
