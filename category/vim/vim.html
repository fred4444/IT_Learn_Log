大小写

查看帮助是:
:h ~
:h gU
:h gu
单字符转换, 在普通模式下在要转换大小写的字母上按~, 光标会移动到下一个字处
范围转换, 先按g再按
~大小写转换
u转换为小写
U转换为大写
再选择范围, 范围的选择同y, d等的方式, 如将2个word转换为大写则按gU2w, 从当前到行末转换为小写则为gu$

g~~或g~g~将整行转换
gUU或gUgU将整行转换为大写
guu或gugu将整行转换为小写

To turn one line into title caps, make every first letter of a word uppercase:
:s/\v<(.)(\w*)/\u\1\L\2/g

列编辑

按ctrl+v进入visual block可视模式按上,下或j,k可以选择这些列所包含的行, 然后可以进行删除等操作, 如按d删除选中的区域
若要多列插入相同的文本则需要在按ctrl+v进入可视模式后选择好相应的行后按shift+i或shift+I, 然后输入要插入的内容, 此时只用一行显示插入的内容, 输入完后按两次Esc键则在此列的多行均插入了输入的内容

ctrl+s, ctrl+q

在linux系统下的vim按下ctrl+s会导致被锁住, 不能动了, 要恢复按ctrl+q
ctrl+s在linux中的作用是停止向终端输出
ctrl+q在linux中的作用是恢复向终端输出流
调试程序时程序不断的输出信息, 可以用crtl+s锁住当前屏幕的信息, 用完后再用crtl+q恢复

寄存器

查看帮助是:
:help reg
:help registers
There are ten types of registers: *registers* *E354*
1. The unnamed register ""
2. 10 numbered registers "0 to "9
3. The small delete register "-
4. 26 named registers "a to "z or "A to "Z
5. three read-only registers ":, "., "%
6. alternate buffer register "#
7. the expression register "=
8. The selection and drop registers "*, "+ and "~
9. The black hole register "_
10. Last search pattern register "/

1. Unnamed register "" *quote_quote* *quotequote*
Vim fills this register with text deleted with the "d", "c", "s", "x" commands or copied with the yank "y" command, regardless of whether or not a specific register was used (e.g. "xdd). This is like the unnamed register is pointing to the last used register. Thus when appending using an uppercase register name, the unnamed register contains the same text as the named register.
An exception is the '_' register: "_dd does not store the deleted text in any register.
Vim uses the contents of the unnamed register for any put command (p or P) which does not specify a register. Additionally you can access it with the name '"'. This means you have to type two double quotes. Writing to the "" register writes to register "0.
{Vi: register contents are lost when changing files, no '"'}

2. Numbered registers "0 to "9 *quote_number* *quote0* *quote1* *quote2* *quote3* *quote4* *quote5* *quote6* *quote7* *quote8* *quote9*
Vim fills these registers with text from yank and delete commands.
Numbered register 0 contains the text from the most recent yank command, unless the command specified another register with ["x].
Numbered register 1 contains the text deleted by the most recent delete or change command, unless the command specified another register or the text is less than one line (the small delete register is used then). An exception is made for the delete operator with these movement commands: |%|, |(|, |)|, |`|, |/|, |?|, |n|, |N|, |{| and |}|. Register "1 is always used then (this is Vi compatible). The "- register is used as well if the delete is within a line. Note that these characters may be mapped. E.g. |%| is mapped by the matchit plugin.
With each successive deletion or change, Vim shifts the previous contents of register 1 into register 2, 2 into 3, and so forth, losing the previous contents of register 9.
{Vi: numbered register contents are lost when changing files; register 0 does not exist}

3. Small delete register "- *quote_-* *quote-*
This register contains text from commands that delete less than one line, except when the command specifies a register with ["x].
{not in Vi}

4. Named registers "a to "z or "A to "Z *quote_alpha* *quotea*
Vim fills these registers only when you say so. Specify them as lowercase letters to replace their previous contents or as uppercase letters to append to their previous contents. When the '>' flag is present in 'cpoptions' then a line break is inserted before the appended text.

5. Read-only registers ":, ". and "%
These are '%', '#', ':' and '.'. You can use them only with the "p", "P", and ":put" commands and with CTRL-R. {not in Vi}
*quote_.* *quote.* *E29*
". Contains the last inserted text (the same as what is inserted with the insert mode commands CTRL-A and CTRL-@). Note: this doesn't work with CTRL-R on the command-line. It works a bit differently, like inserting the text instead of putting it ('textwidth' and other options affect what is inserted).
*quote_%* *quote%*
"% Contains the name of the current file.
*quote_:* *quote:* *E30*
": Contains the most recent executed command-line. Example: Use "@:" to repeat the previous command-line command.
The command-line is only stored in this register when at least one character of it was typed. Thus it remains unchanged if the command was completely from a mapping.
{not available when compiled without the |+cmdline_hist| feature}

6. Alternate file register "# *quote_#* *quote#*
Contains the name of the alternate file for the current window. It will change how the |CTRL-^| command works.
This register is writable, mainly to allow for restoring it after a plugin has changed it. It accepts buffer number: >
let altbuf = bufnr(@#)
...
let @# = altbuf
It will give error |E86| if you pass buffer number and this buffer does not exist.
It can also accept a match with an existing buffer name: >
let @# = 'buffer_name'
Error |E93| if there is more than one buffer matching the given name or |E94| if none of buffers matches the given name.

7. Expression register "= *quote_=* *quote=* *@=* This is not really a register that stores text, but is a way to use an expression in commands which use a register. The expression register is read-write. When typing the '=' after " or CTRL-R the cursor moves to the command-line, where you can enter any expression (see |expression|). All normal command-line editing commands are available, including a special history for expressions. When you end the command-line by typing <CR>, Vim computes the result of the expression. If you end it with <Esc>, Vim abandons the expression. If you do not enter an expression, Vim uses the previous expression (like with the "/" command). The expression must evaluate to a String. A Number is always automatically converted to a String. For the "p" and ":put" command, if the result is a Float it's converted into a String. If the result is a List each element is turned into a String and used as a line. A Dictionary or FuncRef results in an error message (use string() to convert). If the "= register is used for the "p" command, the String is split up at <NL> characters. If the String ends in a <NL>, it is regarded as a linewise register. {not in Vi} 8. Selection and drop registers "*, "+ and "~ Use these registers for storing and retrieving the selected text for the GUI. See |quotestar| and |quoteplus|. When the clipboard is not available or not working, the unnamed register is used instead. For Unix systems the clipboard is only available when the |+xterm_clipboard| feature is present. {not in Vi} Note that there is only a distinction between "* and "+ for X11 systems. For an explanation of the difference, see |x11-selection|. Under MS-Windows, use of "* and "+ is actually synonymous and refers to the |gui-clipboard|. *quote_~* *quote~* *<Drop>* The read-only "~ register stores the dropped text from the last drag'n'drop operation. When something has been dropped onto Vim, the "~ register is filled in and the <Drop> pseudo key is sent for notification. You can remap this key if you want; the default action (for all modes) is to insert the contents of the "~ register at the cursor position. {not in Vi} {only available when compiled with the |+dnd| feature, currently only with the GTK GUI} Note: The "~ register is only used when dropping plain text onto Vim. Drag'n'drop of URI lists is handled internally. 9. Black hole register "_ *quote_* When writing to this register, nothing happens. This can be used to delete text without affecting the normal registers. When reading from this register, nothing is returned. {not in Vi} 10. Last search pattern register "/ *quote_/* *quote/* Contains the most recent search-pattern. This is used for "n" and 'hlsearch'. It is writable with `:let`, you can change it to have 'hlsearch' highlight other matches without actually searching. You can't yank or delete into this register. The search direction is available in |v:searchforward|. Note that the value is restored when returning from a function |function-search-undo|. {not in Vi} You can write to a register with a `:let` command |:let-@|. Example: > :let @/ = "the" If you use a put command without specifying a register, Vim uses the register that was last filled (this is also the contents of the unnamed register). If you are confused, use the `:dis` command to find out what Vim will put (this command displays all named and numbered registers; the unnamed register is labelled '"').
查看所有寄存器中的内容:
:reg
"无名寄存器一个双引号, 这个寄存器是一个默认寄存器, 复制, 删除等操作都将内容放入此寄存器中, 如:y,dd,D,S,s,X,x,C,c等操作, 在没有指定寄存器名称时就默认使用此寄存器
0复制寄存器, 只有复制命令的内容放到此寄存器中, 如y和yy, 用"0p粘贴
1到9删除寄存器, 只有删除的内容放到此寄存器中, 如d和c, 最新删除的为1, 每次删除从8到1依次将内容覆盖到下一个编号的寄存器中, 如用8寄存器覆盖9寄存器, 用1寄存器覆盖2寄存器, 再将最新删除的内容放到1寄存器中, 如要用上次删除的内容用"2p粘贴
a到z命名寄存器, 指定使用的寄存器, 如将当前行复制到m寄存器中则再普通模式中按"myy其中"m用来指定寄存器, 大写的A到Z为追加, 如"Myy则为复制当前行的内容并追加到m寄存器中
_黑洞寄存器, 指定使用_寄存器的内容不会放到其它寄存器中
-行内删除寄存器, 保存在一行之内删除的内容, 若在删除时指定寄存器则不会存入此寄存器中
.只读寄存器, 保存最后插入的文本
%只读寄存器, 当前编辑的文件的文件名
:只读寄存器, 保存最后一次的命令行, 如用:输入的命令或宏等
/搜索寄存器, 保存上次搜索的内容
#交替文件寄存器, 保存刚才编辑的文件的文件名
=表达式寄存器, 在命令行里提示=，按需输入表达式，结果将会保存到=寄存器中

不同操作系统下行尾换行符的差异^M

unix及linux下文本文件的换行符为\n
dos及windows下文本文件的换行符为\r\n
\r 的值为 13 在16进制表示法中为0x0b
\n 的值为 10 在16进制表示法中为0x0a

将整个文件的换行符从\r\n转换为\n可以在 bash shell 中调用
dos2unix filename
filename为相应的文件名
将整个文件的换行符从\n转换为\r\n可以在 bash shell 中调用
unix2dos filename
filename为相应的文件名
在vim中通过
set fileformat=unix
自动将当前文件的换行符替换为unix风格的\n
也可使用简写形式
set ff=unix
若转换为dos风格的换行符则
set ff=dos

在vim中可以通过查看16进制格式来查看换行符的情况
在bash shell中可用如下命令来查看哪些文件含有^M
grep "^M" -lr .
其中^M=ctrl+v,ctrl+m即先按ctrl+v再按ctrl+m
多文件的批量转换可以参考sed命令
sed -e "s/^V^M//" oldfilename > newfilename
或cat命令
cat oldfilename | tr -d "^V^M" > newfilename
其中^V=ctrl+v, ^M=ctrl+m, ^V^M即先按ctrl+v再按ctrl+m

含有^M的文件即以\r\n为换行符的文件直接用vim打开是看不见^M的因为用vim打开文件此类文件时默认的fileformat是dos,当设置为unix时会将原有的\r\n替换为\n
可以在打开文件时加上 -b 参数就可以看见^M了
vim -b filename
filename为相应的文件名
在此情况下可以替换换行符
:%s/\r//g
or
:%s/^M//g
其中^M=ctrl+v,ctrl+m即先按ctrl+v再按ctrl+m

折叠模式
帮助为 :help fdm
有6种模式
manual : Folds are created manually. : 手工创建折叠, 默认值, 用:mkview保存折叠信息, 用:loadview加载折叠信息, 保存的折叠信息在~/.vim/view/目录下有一个用=+代替/的对应全路径文件名的文件, 因此若文件移动位置或改名会导致找不到原来的折叠信息文件
indent : Lines with equal indent form a fold. : 等量的缩进作为折叠
expr : 'foldexpr' gives the fold level of a line. : 通过表达式定义缩进级别
marker : Markers are used to specify folds. : 标志指定折叠默认的折叠标志是{{{和}}}
syntax : Syntax highlighting items specify folds. : 语法高亮指定缩进
diff : Fold text that is not changed. : 未改变的文本作为缩进
设置折叠模式
:set foldmethod=manual
or
:set fdm=manual

关闭折叠

打开折叠

打开折叠和关闭折叠
zo打开当前折叠一层
zO当前折叠递归打开,直到嵌套的最后一层子折叠
zc关闭当前折叠一层
zC当前折叠递归关闭,从当前层级的折叠到最顶层的折叠, 不包含当前层的子折叠
za为zo与zc自动切换执行
zA为zO与zC自动切换执行
zr其中的r意为reduce, 作用是将变量foldlevel减1, 并将对应层级的折叠打开
zR递归执行zr直到foldlevel为0, 展开所有层的折叠
zm其中的m意为more, 作用是将变量foldlevel加1, 并将对应层级的折叠关闭
zM递归执行zm直到所有层的折叠
:set foldlevel=0直接指定foldlevel, 并相应的打开或关闭相应层级的折叠
zx关闭除使当前行可见的所有折叠, 即将当前行所在的那层折叠到其所有的直属父系折叠打开, 其余的折叠都关闭, 包括当前折叠的所有子折叠. 下面两行是帮助中写的及其翻译可参考
更新折叠：撤消手动打开和关闭的折叠：重新应用“foldlevel”，然后执行“zv”：查看光标行。也会强制重新计算折叠。当使用“foldexpr”并且缓冲区更改的方式导致折叠无法正确更新时，这很有用。
Update folds: Undo manually opened and closed folds: re-apply 'foldlevel', then do "zv": View cursor line. Also forces recomputing folds. This is useful when using 'foldexpr' and the buffer is changed in a way that results in folds not to be updated properly.

用命名缓冲区复制粘贴

在vim中有26个命名缓冲区名字为a~z(注意这26个命名缓存区与其它操作的命名缓冲区是共享的如在a缓冲区中录了一个宏，然后又将复制的值放到了a缓冲区中则原先的宏的内容被覆盖)在删除和复制时可以指定缓冲区将值暂存在指定的缓冲区中，在粘贴时在通过指定缓冲区名称来粘贴指定缓冲区中的内容。

复制到命名缓冲区
举例，将当前光标所在位置及其后共3行内容复制到命名缓冲区a
按ESC进入命令模式，按"a3yy

在命令行模式中用
起始行号, 末尾行号 yank 缓冲区名称
将指定行范围的内容复制到指定名称的缓冲区中
举例，将10到20的内容复制到命名缓冲区a中
:10, 20 yank a
若不指定缓冲区名称则复制到默认缓冲区中不在这26个命名缓冲区中
举例，复制10到20的内容
:10, 20 yank

从命名缓冲区中粘贴
举例，将命名缓冲区a中的内容粘贴到当前光标所在位置
按ESC进入命令模式，按"ap

录制和重放宏

在vim中有26个命名缓冲区名字为a~z(注意这26个命名缓存区与其它操作的命名缓冲区是共享的如在a缓冲区中录了一个宏，然后又将复制的值放到了a缓冲区中则原先的宏的内容被覆盖)宏可以录制一组操作，然后重放宏，在自动重复做这种操作一次或多次。
录制宏
举例，将宏录制到命名缓冲区a中。
按ESC进入命令模式，按q再按a然后在左下角会出现recording表明处于录制状态，在这之后我们做的操作将会被录到命名缓冲区a中，操作做完后按ESC进入命令模式，按q结束录制。
重放宏
举例，重放命名缓冲区a中的宏。
按ESC进入命令模式，按@再按a。
相关帮助为:
:h q
:h @
查看宏代码, 保存宏代码, 自动加载宏
帮助为:
:h let-@
先录制一个宏按qa开始录制宏并将宏放置在a缓冲区中
再按a开始输入文字this is the macro test text
按esc键再q结束宏的录制.
基于vim中命名缓冲区的内容是共享的, 用"ap将a缓冲区中的代码粘贴出来即可查看宏代码内容为:
athis is the macro test text^[
若欲将这个宏保存到m共享缓冲区中, 并在每次启动时都生效, 可将内容写入到.vimrc文件中并将其赋给m缓冲区即写入let @m = "内容", 本例为
let @m = "athis is the macro test text^["
如此则每次启动vim时都会自动加载此宏到m缓冲区中, 用@m即可调用此宏
