不同操作系统下行尾换行符的差异^M

unix及linux下文本文件的换行符为\n
dos及windows下文本文件的换行符为\r\n
\r 的值为 13 在16进制表示法中为0x0b
\n 的值为 10 在16进制表示法中为0x0a

将整个文件的换行符从\r\n转换为\n可以在 bash shell 中调用
dos2unix filename
filename为相应的文件名
将整个文件的换行符从\n转换为\r\n可以在 bash shell 中调用
unix2dos filename
filename为相应的文件名
在vim中通过
set fileformat=unix 自动将当前文件的换行符替换为unix风格的\n
也可使用简写形式
set ff=unix
若转换为dos风格的换行符则
set ff=dos

在vim中可以通过查看16进制格式来查看换行符的情况
在bash shell中可用如下命令来查看哪些文件含有^M
grep "^M" -lr .
其中^M=ctrl+v,ctrl+m即先按ctrl+v再按ctrl+m
多文件的批量转换可以参考sed命令

含有^M的文件即以\r\n为换行符的文件直接用vim打开是看不见^M的因为用vim打开文件此类文件时默认的fileformat是dos,当设置为unix时会将原有的\r\n替换为\n
可以在打开文件时加上 -b 参数就可以看见^M了
vim -b filename
filename为相应的文件名
在此情况下可以替换换行符
:%s/\r//g
or
:%s/^M//g
其中^M=ctrl+v,ctrl+m即先按ctrl+v再按ctrl+m

折叠模式
帮助为 :help fdm
有6种模式
manual : Folds are created manually. : 手工创建折叠, 默认值, 用:mkview保存折叠信息, 用:loadview加载折叠信息, 保存的折叠信息在~/.vim/view/目录下有一个用=+代替/的对应全路径文件名的文件, 因此若文件移动位置或改名会导致找不到原来的折叠信息文件
indent : Lines with equal indent form a fold. : 等量的缩进作为折叠
expr : 'foldexpr' gives the fold level of a line. : 通过表达式定义缩进级别
marker : Markers are used to specify folds. : 标志指定折叠默认的折叠标志是{{{和}}}
syntax : Syntax highlighting items specify folds. : 语法高亮指定缩进
diff : Fold text that is not changed. : 未改变的文本作为缩进
设置折叠模式
:set foldmethod=manual
or
:set fdm=manual


创建折叠
:line_number, line_number fold
关闭折叠
:line_number normal zc
打开折叠
:line_number normal zo

创建折叠    (仅当foldmethod设为manual或marker时有效)
zf10G创建从当前行到当前文件第10行的折叠
8zf或8zf+或8zfj或8zf↓或zf8+或zf8j或zf8↓从当前行及下面8行创建折叠(此折叠共9行)
        8zf-或8zfk或8zf↓或zf8+或zf8j或zf8↑从当前行及上面8行创建折叠(此折叠共9行)
zf%从当前行到与当前行对应的括号创建折叠如(),[],{}等, 如当前行含有{则会找到对应层级的}
按shift+v或输入V(大写的v)进入visual模式上下选择折叠的范围然后按zf则将选择的行创建为折叠
在要创建折叠的起始行, 在命令模式下按zF, 然后输入内容, 换行用回车, 则连续多行为创建的折叠, 若用o创建的新行则此次折叠输入结束


打开折叠和关闭折叠
zo打开当前折叠一层
zO当前折叠递归打开,直到嵌套的最后一层子折叠
zc关闭当前折叠一层
zC当前折叠递归关闭,从当前层级的折叠到最顶层的折叠, 不包含当前层的子折叠
za为zo与zc自动切换执行
zA为zO与zC自动切换执行
zr其中的r意为reduce, 作用是将变量foldlevel减1, 并将对应层级的折叠打开
zR递归执行zr直到foldlevel为0, 展开所有层的折叠
zm其中的m意为more, 作用是将变量foldlevel加1, 并将对应层级的折叠关闭
zM递归执行zm直到所有层的折叠
:set foldlevel=0直接指定foldlevel, 并相应的打开或关闭相应层级的折叠
zx关闭除使当前行可见的所有折叠, 即将当前行所在的那层折叠到其所有的直属父系折叠打开, 其余的折叠都关闭, 包括当前折叠的所有子折叠. 下面两行是帮助中写的及其翻译可参考
更新折叠：撤消手动打开和关闭的折叠：重新应用“foldlevel”，然后执行“zv”：查看光标行。也会强制重新计算折叠。当使用“foldexpr”并且缓冲区更改的方式导致折叠无法正确更新时，这很有用。
Update folds: Undo manually opened and closed folds: re-apply 'foldlevel', then do "zv": View cursor line. Also forces recomputing folds. This is useful when using 'foldexpr' and the buffer is changed in a way that results in folds not to be updated properly.

用命名缓冲区复制粘贴

在vim中有26个命名缓冲区名字为a~z(注意这26个命名缓存区与其它操作的命名缓冲区是共享的如在a缓冲区中录了一个宏，然后又将复制的值放到了a缓冲区中则原先的宏的内容被覆盖)在删除和复制时可以指定缓冲区将值暂存在指定的缓冲区中，在粘贴时在通过指定缓冲区名称来粘贴指定缓冲区中的内容。

复制到命名缓冲区
举例，将当前光标所在位置及其后共3行内容复制到命名缓冲区a
按ESC进入命令模式，按"a3yy

在命令行模式中用
起始行号, 末尾行号 yank 缓冲区名称
将指定行范围的内容复制到指定名称的缓冲区中
举例，将10到20的内容复制到命名缓冲区a中
:10, 20 yank a
若不指定缓冲区名称则复制到默认缓冲区中不在这26个命名缓冲区中
举例，复制10到20的内容
:10, 20 yank

从命名缓冲区中粘贴
举例，将命名缓冲区a中的内容粘贴到当前光标所在位置
按ESC进入命令模式，按"ap

录制和重放宏

在vim中有26个命名缓冲区名字为a~z(注意这26个命名缓存区与其它操作的命名缓冲区是共享的如在a缓冲区中录了一个宏，然后又将复制的值放到了a缓冲区中则原先的宏的内容被覆盖)宏可以录制一组操作，然后重放宏，在自动重复做这种操作一次或多次。
录制宏
举例，将宏录制到命名缓冲区a中。
按ESC进入命令模式，按q再按a然后在左下角会出现recording表明处于录制状态，在这之后我们做的操作将会被录到命名缓冲区a中，操作做完后按ESC进入命令模式，按q结束录制。
重放宏
举例，重放命名缓冲区a中的宏。
按ESC进入命令模式，按@再按a。
相关帮助为:
:h q
:h @
查看宏代码, 保存宏代码, 自动加载宏
帮助为:
:h let-@
先录制一个宏按qa开始录制宏并将宏放置在a缓冲区中
再按a开始输入文字this is the macro test text
按esc键再q结束宏的录制.
基于vim中命名缓冲区的内容是共享的, 用"ap将a缓冲区中的代码粘贴出来即可查看宏代码内容为:
athis is the macro test text^[
若欲将这个宏保存到m共享缓冲区中, 并在每次启动时都生效, 可将内容写入到.vimrc文件中并将其赋给m缓冲区即写入let @m = "内容", 本例为
let @m = "athis is the macro test text^["
如此则每次启动vim时都会自动加载此宏到m缓冲区中, 用@m即可调用此宏
