文件内容格式化

快捷键
Windows 下为 Shift + Alt + F
Ubuntu 下为 Shift + Ctrl + I
Mac 下为 Shift + Option + F


自动格式化设置
在设置中搜索 emmet 或 emmet.include
然后在 "Emmet: Include Languages" 这项中增加相应要自动格式化的文件后缀的项, 如json文件则 item 为 *.json, value 为 json

然后在配置文件 setting.json 中添加配置项
"editor.formatOnType": true,
"editor.formatOnSave": true,

折叠 folder

选择 File -> Preferences -> Keyboard Shortcuts 打开快捷键设置面板
搜索 fold 可以看到相关快捷键, 如
Fold Recursively Ctrl + k Ctrl + [
即递归折叠快捷键为先按 Ctrl + k 然后在按 Ctrl + [
Unfold Recursively Ctrl + k Ctrl + ]
即递归展开折叠快捷键为先按 Ctrl + k 然后在按 Ctrl + ]
Fold All Ctrl + k Ctrl + 0
即全部折叠快捷键为先按 Ctrl + k 然后在按 Ctrl + 0
Unfold All Ctrl + k Ctrl + J
即全部展开折叠快捷键为先按 Ctrl + k 然后在按 Ctrl + J

配置 git bash

若是安装了 git bash 但仍找不到 git bash 可以自行配置
选择 File -> Preferences -> Settings
打开配置面板, 在 User 配置标签卡下选择 Text Editor 下的 Font 然后在右侧找到 Font Weight 这项, 这项下有一行 Edit in Settings.json 点击这项可以进入到用户基本的json配置文件
在其中输入
"terminal.integrated.profiles.windows":
这项, 然后会自动补全内容, 在这里新增一项, 这里以起名为 "Bash" 为例说明整个配置文件的内容
"terminal.integrated.profiles.windows": {
"Bash": {
"path": [
"Git路径\\Git\\bin\\bash.exe",
"Git路径\\Git\\git-bash.exe",
],
"args": [],
"icon" : "terminal-bash"
},
"Git Bash": {
"path": [
"Git路径\\Git\\git-bash.exe",
],
"args": [],
"icon" : "terminal-bash"
},
"PowerShell": {
"source": "PowerShell",
"icon": "terminal-powershell"
},
"Command Prompt": {
"path": [
"${env:windir}\\Sysnative\\cmd.exe",
"${env:windir}\\System32\\cmd.exe"
],
"args": [],
"icon": "terminal-cmd"
},
},
"terminal.integrated.defaultProfile.windows": "Bash",
其中将“Git路径”换为相应的路径字符串
最后一行
"terminal.integrated.defaultProfile.windows": "Bash",
为设置默认的终端, 另外要清楚workspace与Folder标签页对应的 settings.json 中的这些配置内容, 然后重启 vscode
有两点要注意, 这里有一个名为 "Bash" 和一个名为 "Git Bash", 这个名为"Git Bash"的不起作用即不能使用这个名字, 所以才换了一个名字即不能使用"Git Bash"这个名字, 另一点是路径要使用 Git\\bin\\bash.exe 而不能使用 Git\\git-bash.exe, 这个会导致单启一个分离的窗口无法整合进去
